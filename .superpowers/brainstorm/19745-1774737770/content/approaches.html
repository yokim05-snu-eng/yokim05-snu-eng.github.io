Site Architecture: 2 Approaches

Both use minimal custom HTML/CSS on Jekyll. The difference is in page structure and navigation style.

A

Single-Page App Feel

Homepage is the entire experience. The 5 nav buttons scroll to sections on the same page (anchor links). Each "page" is a full-viewport section: Hero → Statement → 발전계획서 → 실적 → 초심 → About. The nav bar highlights the current section as you scroll.

Pros

  • Extremely simple — one page, one flow
  • Mobile-friendly scrolling experience
  • Easy to maintain — everything in one file
  • Modern, clean feel

Cons

  • 발전계획서 (15 subsections) could make the page very long
  • About page with full CV may feel cramped as a section
  • Harder to link directly to specific content
B

Minimal Multi-Page (Recommended)

Homepage is the hero + nav buttons. Each button leads to its own clean page. 6 total pages: Home, Statement, 발전계획서, 실적, 초심, About. Shared minimal header/footer. Each page is focused and self-contained.

Pros

  • Each page can breathe — 발전계획서 gets its full long-scroll treatment
  • About page works naturally as a full CV
  • Direct URLs for sharing specific pages
  • Homepage stays punchy — hero text + 5 buttons, done
  • Still very simple — just 6 HTML pages with shared layout

Cons

  • Slightly more files to manage (still only 6)
  • Page transitions instead of smooth scrolling
